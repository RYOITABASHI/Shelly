## 素材1: ヒーローGIF（12秒）

**一番大事な素材。README冒頭に載る。**

### コンセプト
> 「Terminal にエラー → Chat で一言 → AIが修正コード提案 → ワンタップで実行 → 自動セーブ」

### 事前準備
1. Z Fold6を展開、マルチペインモード（左: Chat、右: Terminal）
2. 右Terminalでわざとエラーを出しておく:
   ```
   $ npm run build
   Error: Cannot find module './utils'
   ```
   （スクショにあったやつがそのまま使える）
3. 左Chatは空（New Chat状態）
4. キーボードは **隠した状態** で開始

### 撮影手順（録画ONにしてから）

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-2秒 | **何もしない**。エラー画面を見せる | 右ペインに赤いエラー表示 |
| 2-4秒 | 左Chat入力欄をタップ → キーボード出現 | Nacreキーボードがチラ見え |
| 4-6秒 | `fix the error on the right` と入力 | 入力テキストが見える |
| 6秒 | **送信ボタン**（▶）をタップ | メッセージが送信される |
| 6-9秒 | **待つ**。AIが応答する | AI応答がストリーミング表示 → ActionBlock出現 |
| 9-10秒 | ActionBlockの **[▶ Run]** をタップ | Terminal側でコマンド実行 |
| 10-12秒 | **待つ**。実行完了 + 💾セーブバッジ | 右TerminalにFix適用 + 💾表示 |

### NG集・リカバリー
- AI応答が遅い場合: Cerebras/Groq経由のLocal LLMだと速い
- ActionBlockが画面外: スクロールしてからタップ
- セーブバッジが出ない: 2秒待って出なければそのまま録画停止でOK

### 後処理の字幕（ffmpegで焼き込み）
```
0-3秒: "The copy-paste problem ends here."
4-8秒: "Say it. AI reads the terminal. Suggests a fix."
8-12秒: "One tap. It runs. Auto-saved."
```

## 素材2: シングルペイン クロスペイン（10秒）

### コンセプト
> 「折りたたみでもクロスペインは動く。Terminalのエラーを覚えてて、Chatで直せる」

### 事前準備
1. Z Fold6を **折りたたむ** or シングルペインモード
2. **言語を日本語に切替**（Settings → 言語）
3. Terminalタブでエラーを出しておく:
   ```
   $ python test.py
   AssertionError: expected 200, got 404
   ```
4. Terminalタブが表示された状態で開始

### 撮影手順

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-2秒 | **何もしない**。Terminalのエラーを見せる | エラーメッセージ |
| 2-3秒 | **Chatタブ**をタップして切替 | Chat画面に遷移 |
| 3-5秒 | 入力欄タップ → `さっきのエラー直して` と入力 | 日本語入力（Nacre映る） |
| 5秒 | **送信** | メッセージ送信 |
| 5-8秒 | **待つ**。AI応答 | クロスペイン参照でTerminal内容に基づいた修正提案 |
| 8-9秒 | **[▶ Run]** をタップ | 修正コマンド実行 |
| 9-10秒 | **待つ** | 完了表示 |

### ポイント
- **日本語で指示 → AIが理解** というのが伝わるように
- 「さっきの」= Terminalタブの内容をAIが覚えてる証拠

## 素材3: @team マルチAI合議（15秒）

### コンセプト
> 「@team で複数AIに同時質問 → 各自が回答 → ファシリテーターが統合」

### 事前準備
1. シングルペイン、Chatタブ
2. **言語を英語に戻す**
3. 複数AIプロバイダー設定済み（最低2つ）
4. New Chat状態

### 撮影手順

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-2秒 | 入力欄タップ → `@team What architecture should I use for a REST API?` と入力 | @teamが色付きで表示 |
| 2-3秒 | **送信** | メッセージ送信 |
| 3-10秒 | **待つ**。各AIが順次応答 | 🟡Claude、🔵Gemini等がカラーバッジ付きで表示 |
| 10-13秒 | **待つ**。ファシリテーターサマリー | 🎙️ファシリテーターが統合回答 |
| 13-15秒 | 軽くスクロールして全体を見せる | 合議の全体像 |

### ポイント
- 各AIの回答がリアルタイムで追加される「レース感」が見どころ
- ファシリテーターの統合サマリーが出たら少し止めて読ませる

## 素材4: 自動セーブポイント + 元に戻す（10秒）

### コンセプト
> 「AIが変更 → 自動セーブ → diff確認 → ワンタップでUndo」

### 事前準備
1. シングルペイン、Chatタブ
2. AI操作後のSavepointBubbleが表示されている状態
   - 「📁 Modified N files」+ [Undo] + [View changes]
3. 表示されてなければ: 何かAIにファイル編集させてセーブポイント作る

### 撮影手順

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-1秒 | **何もしない**。SavepointBubbleを見せる | 💾バブル表示 |
| 1-2秒 | **[View changes]** をタップ | DiffViewerモーダルが開く |
| 2-5秒 | **diffを眺める**。軽くスクロール | 緑(追加)/赤(削除)のハイライト |
| 5-6秒 | モーダルを **閉じる**（戻るボタン or 外側タップ） | Chat画面に戻る |
| 6-8秒 | **[Undo]** をタップ | 元に戻す実行 |
| 8-10秒 | **待つ** | 「Reverted!」的な確認表示 |

## 素材5: CLI Co-Pilot リアルタイム翻訳（10秒）

### コンセプト
> 「TerminalでClaude Code実行 → Chat側にリアルタイム翻訳が出る」

### 事前準備
1. Z Fold6を展開、マルチペインモード（左: Chat、右: Terminal）
2. Cerebras/Groq APIキー設定済み（翻訳速度確保）
3. 右Terminalで `claude` と打てる状態

### 撮影手順

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-2秒 | 右Terminalで `claude` と入力しEnter | claude起動 |
| 2-4秒 | claudeに何か指示（例: `explain this project`） | claude応答開始 |
| 4-7秒 | **左Chatを見る** | 翻訳オーバーレイがリアルタイム表示 |

## 素材6: GitHub連携 + 自動チェック（15秒）

### コンセプト
> 「未pushのセーブポイント → AIが同期提案 → ワンタップpush → CI自動設定」

### 事前準備

2. GitHub PAT設定済み
3. プロジェクトにgit remote originあり
4. 5件以上の未pushセーブポイント

### 撮影手順

| 秒数 | 操作 | 画面に映るもの |
|------|------|--------------|
| 0-2秒 | Sync提案バブルを見せる | 「💡 N savepoints not synced. Sync to GitHub?」 |
| 2-3秒 | **[Sync]** をタップ | push開始 |
| 3-6秒 | **待つ** | プログレス → 「Synced!」 |
| 6-9秒 | **待つ**。自動チェック提案が出る | 「✓ Auto-check available」+ [Turn on] |
| 9-10秒 | **[⚡ Turn on]** をタップ | CI設定開始 |
| 10-13秒 | **待つ** | 「Auto-check is on!」 |
| 13-15秒 | **ホールド** | 完了状態 |

## Tips

- **テイクは何度でもOK**。一番良いのだけ残す
- **入力は落ち着いて**。速すぎるとタイポが映る
- **AI応答が遅い時**: 録画を止めて、応答が出てから再録画 → 後で繋ぐ手もある
- **通知が映り込んだ**: おやすみモード確認。映り込んだらそのテイクは破棄
- **キーボードの見せ方**: 入力時にNacreが自然に映る。無理に見せなくてOK
- **録画停止のタイミング**: 最後の操作から1-2秒待ってから停止。余白があると編集しやすい
