### 7. ターミナル出力の等幅問題

**背景**: Silkscreen は非等幅なので `ls -la` の列がモックでもズレているが、**モックのズレと実機のズレが完全一致しない**可能性がある。

**対処**:
- 実機で mock-1 / mock-5 の `ls -la` と実機の `ls -la` を並べて、列の揃い方を見比べる
- もしユーザーが不満なら `Mono` プリセットに切替可能(既に実装済)
- もしくは **Silkscreen-like な等幅 PX フォント**を別途探して silkscreen スロットに入れ替え
- 候補: **Pixelify Sans Monospaced**(名前の通り)、**TinyUnicode**

**規模**: 小(1 ファイル差し替え、15 分)or 調査のみ

## 重要な設計判断メモ(次セッションが触る時の注意)

1. **theme.config.ts の `colors` は mutable object**。 `const C = { ...colors }` で spread するとプリセット切替で追従しない。
2. **`theme.config.ts` → `./lib/theme-presets`**(相対 import)。`@/` エイリアスは Metro bundle の順序問題で theme.config.ts では解決失敗する。
3. **`theme-presets.ts` → `require('@/theme.config')`**(lazy)。循環依存回避のため `require()` を使っている。静的 import に戻さないこと。
4. **Silkscreen は非等幅**。ターミナルで列が揃わない。モックも揃っていないので意図通り。
5. **`settings.uiFont` は 4 値**: `shelly` / `silkscreen` / `pixel` / `mono`。これ以外を想定したコードは書かないこと。
6. **PTY セッションは JS の theme state に依存しない**。ネイティブ側で生きてるので、ShellLayout 再マウントで session id は保持される。
7. **TerminalHeader / SessionInfoBar は削除済**。import しようとしないこと。
8. **pane-type pill と CLI タブは別物**。前者は pane type(terminal/ai/browser...)、後者は pane 内の session 切替。
