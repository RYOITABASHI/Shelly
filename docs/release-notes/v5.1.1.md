# Shelly v5.1.1 — Security diagnostics + build 782 refresh

Patch release for the post-v5.1.0 hardening work verified on build 782.

## Highlights

- Added `shelly-doctor` credential hygiene checks:
  - warns when `/sdcard/Download` still contains credential handoff files
  - verifies private-mode credential files such as `.claude.json`,
    `.credentials.json`, and `.codex/auth.json`
  - reports whether common API-key environment variables are present
- Added app debug-log redaction for common API key and token patterns before
  writing to logcat.
- Expanded `SECURITY.md` and README with the explicit Shelly security model:
  Android app sandbox, not a hardened VM; AI agents can read/write what the app
  can access once approved.
- Fixed the restored-terminal startup path so rehydrated sessions reliably start
  after install/update.
- Improved the Builds / Updates panel so recent GitHub Actions runs can be
  viewed without `gh auth`, with duration and failed-log surfacing.
- Reworked the redaction unit-test fixture to avoid GitHub secret-scanning
  false positives.

## Notes

This release does not change the credential import model. Claude Code and
Gemini CLI still require credential transplant from another environment for
first-run OAuth. The new `shelly-doctor` checks make that workflow easier to
audit after import.
