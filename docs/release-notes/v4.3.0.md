# Shelly v4.3.0

Second stabilization pass after a week of on-device testing on Samsung
Galaxy Z Fold6 / Android 16. All user-visible regressions from the prior
release are now closed, and the Claude Code Bash tool works out of the box.

## What's new

- **Claude Code 2.1.112 reliable startup + Bash tool** (bug #117). The pinned
  cli.js tier is now the default path, and the Android-specific fixes for
  `child_process.spawn({ shell: true })` let the Bash tool, MCP helper, and
  editor spawns actually run. The latest Bun SEA build is still available
  as an opt-in via `SHELLY_PREFER_MUSL_CLAUDE=1`.
- **Multi-pane keyboard rework**. A single `TerminalImeHostView` owns IMM's
  `mServedView` for the whole Activity, so tapping between panes never
  requires IME migration. Fixes the Samsung OneUI stale-servedView bug
  where typing or Japanese composition could route to the wrong pane after
  a focus switch.
- **Paste routing fix for CLIs and TUIs**. Paste now reads
  `/proc/<bashPid>/task/<bashPid>/children` to distinguish "at bash prompt"
  from "inside Claude / Codex / Gemini / vim / less / zsh / nested bash".
  Two-line paste into Claude Code no longer leaks `[200~` or submits
  line-by-line; Japanese composition no longer wedges with
  `setComposingText on inactive InputConnection`.
- **Gemini CLI Android subprocess hardening**. Three breakage classes
  patched: `shell: true` in the auto-update and GrepTool preflight paths,
  and `execSync` in `commandExists()` which was silently failing every
  editor detection (vim / nano / cursor). `!vim --version` and `!nano
  --version` now return real output.
- **Loud drift detectors for bundled CLI sed patches**. Claude Code
  `shell:!0` and Gemini `shell: true` + `commandExists` patterns now fail
  the CI build AND the daily runtime updater with a descriptive error if
  upstream minified bundles shift shape on a version bump — instead of
  silently shipping an APK with a broken Bash tool or editor detection.

## Security hardening

- LD_PRELOAD exec wrapper (`libexec_wrapper.so`) picks up Codex audit
  fixes: `dlsym` null-safety in every wrapper, `SHELL` env validation
  against a trusted-prefix allowlist (blocks path-injection rewrites of
  `/bin/bash`), `snprintf` truncation guard for `/bin/X` and `/usr/bin/X`
  synthesis.
- Two HIGH-severity items from the same audit (PATH-resolved ELFs
  skipping linker routing; `is_elf` TOCTOU) are tracked in `DEFERRED.md`
  as post-release work — neither triggers on Shelly's current exec paths.

## Verification

Release gate on Samsung Galaxy Z Fold6 / OneUI 6 / Android 16:
basic terminal + CommandKeyBar, multi-pane layout safety, multi-pane
focus tap, Claude Code paste, Claude Bash tool, Gemini `!echo`, Codex
cold start, Gemini `!vim --version`, scrcpy PC keyboard per pane, zsh
paste comm-guard, fold/rotate during active IME, Ctrl-C to foreground
job, large paste into editors, force-stop recovery, SSH remote locale,
three-CLI simultaneous session. Details in
`docs/superpowers/specs/2026-04-22-handoff.md`.

## Known limitations

See `README.md` → Known Limitations. Nothing new compared to v4.2.0
except the following carry-overs worth restating for first-time users:

- APK is 420 MB because tools are bundled, not symlinked to Termux. If
  you already have Termux, Shelly still does not use it.
- Claude Code pinned to 2.1.112 by default; latest Bun SEA is opt-in.
  Newer Claude features may require `SHELLY_PREFER_MUSL_CLAUDE=1`.
- Paste into remote SSH bash is line-by-line (documented bionic bash
  readline behavior).
- First-run OAuth flows for Claude Code and Gemini CLI do not complete
  inside Shelly; transplant credentials from a machine where auth
  works — see README "Bring your own credentials".

## Upgrading

Clean install recommended. Uninstall previous Shelly first so bash
regenerates `.bashrc` from the new `BASHRC_VERSION` and `.shelly-cli`
reinstalls against the new CI-patched bundle.

```
adb install -r shelly-v4.3.0.apk
```
