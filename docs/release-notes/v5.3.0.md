# Shelly v5.3.0 — Supported Claude Code and Codex CLIs, API-backed AI pane

Shelly v5.3.0 narrows the release promise to the parts that are now
device-validated and supportable: real interactive Claude Code and Codex
terminal CLIs on Android, plus an AI Pane/background system powered by
explicit API providers.

Gemini is still present in two forms, but with a clear boundary:
Gemini API is supported wherever the user configures an API key; the
interactive Gemini CLI is Experimental because upstream 0.42.x remains
unstable on Android/musl.

## Release surface

| Surface | Status |
|---|---|
| Claude Code CLI | Supported foreground terminal CLI |
| Codex CLI | Supported foreground terminal CLI |
| AI Pane / background agents | Supported through explicit API providers |
| Gemini API | Supported when configured |
| Gemini CLI | Experimental, hidden from Worktrees and Quick Launch |

## Highlights

- **Claude Code remains a user-controlled CLI.** Shelly can launch and
  maintain Claude Code inside Terminal panes, seed workspace trust, and keep
  credentials in the app-private home directory. Shelly does not drive a
  Claude Code subscription as a hidden background worker.
- **Codex now has a Shelly-owned login path.** Bare `codex` checks
  `~/.codex/auth.json`. If credentials are missing or invalid, Shelly opens
  OpenAI's device-code login in the Browser Pane, writes the auth file, then
  launches the normal Codex TUI. Verified on-device with `codex-exec 0.130.0`
  and GPT-5.5.
- **AI Pane and background agents use explicit API providers.** Gemini API,
  Cerebras, Groq, Perplexity, OpenAI-compatible local servers, and other
  configured API routes are the supported automation path.
- **Gemini CLI is demoted to Experimental.** It remains bundled for
  investigation, but it is removed from Worktrees and CLI Quick Launch.
- **Terminal split focus is fixed.** New terminal tabs created from the pane
  UI now receive focus immediately.

## Why Gemini CLI is not blocking this release

The latest Gemini CLI investigation narrowed the issue, but not enough to
make it a release guarantee. Device testing showed that `gemini --version`
works and authenticated accounts can exist under `~/.gemini`, but the
interactive TUI can blank, render very slowly, or trigger shell-tool
commands that terminate with signal 11. Some failures appear tied to
upstream Android/musl behavior in Gemini CLI 0.42.x rather than a single
Shelly-only bug.

The product choice for v5.3.0 is therefore conservative: ship Gemini API
support, keep Gemini CLI bundled for continued testing, and present Claude
Code plus Codex as the supported CLI experience.

## Upgrade notes

- This release bumps app semver, Expo runtimeVersion, Android
  versionName, and Android versionCode together.
- Expo OTA remains disabled for release APKs. The installed APK is the
  source of truth for JS and CLI harness behavior.
- If Codex asks to sign in after upgrade, run bare `codex` and follow the
  in-app device-code flow. Do not use `codex /login` as the primary path.
